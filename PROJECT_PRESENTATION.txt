                            EXECUTIVE SUMMARY
================================================================================

FireGuard is a modern, real-time fire safety monitoring system that detects 
dangerous conditions and sends instant alerts to users. It combines hardware 
sensors, cloud technology, and smart notifications to protect buildings and 
save lives.

Think of it as a "smart fire alarm" that not only detects fire but also tells 
you exactly where it is and what to do about it.

================================================================================
                          WHAT IS FIREGUARD?
================================================================================

FireGuard is a complete fire safety system with three main parts:

1. HARDWARE (The Sensors)
   - ESP32-S3 microcontroller (small computer)
   - MQ-2 Gas Sensor (detects smoke and gas)
   - Flame IR Sensor (detects fire/heat)
   - DHT11 Temperature & Humidity Sensor
   - Buzzer for local alarm
   
   These sensors are placed in different rooms and constantly monitor the air.

2. CLOUD BACKEND (The Brain)
   - Supabase (database and real-time updates)
   - PostgreSQL (stores all data)
   - Real-time subscriptions (instant updates)
   - Edge Functions (automatic actions)
   
   This is where all the data is collected, analyzed, and decisions are made.

3. WEB APPLICATION (The Dashboard)
   - Next.js 14 + React 18 (modern web framework)
   - Real-time monitoring dashboard
   - Device management
   - Analytics and reports
   - User authentication
   - Settings and customization
   
   This is what users see - a beautiful interface to monitor everything.

================================================================================
                        THRESHOLD MANAGEMENT
================================================================================

IMPORTANT: Thresholds can be changed anytime in Settings!

HOW THRESHOLDS WORK:
1. Each device has configurable warning and danger thresholds
2. When sensor data exceeds a threshold, an alert is created
3. The alert captures the CURRENT threshold values at that moment
4. Thresholds are stored in the DEVICE_SETTINGS table
5. Changing thresholds affects FUTURE alerts only, not past ones

CHANGING THRESHOLDS:
1. Go to Settings page
2. Click "Alert Thresholds" tab
3. Adjust warning and danger levels for:
   â€¢ Gas (PPM)
   â€¢ Temperature (Â°C)
   â€¢ Humidity (%)
4. Click "Save Thresholds"
5. New thresholds apply immediately to future alerts

ALERT HISTORY:
â€¢ Past alerts show the thresholds that were active when they were created
â€¢ You can view what threshold triggered each alert
â€¢ This helps track when you changed settings
â€¢ Useful for auditing and compliance

EXAMPLE:
â€¢ Original threshold: Gas danger = 500 PPM
â€¢ Alert created: "Gas level 520 PPM" (shows 500 PPM threshold)
â€¢ You change threshold to: 400 PPM
â€¢ Next alert: "Gas level 420 PPM" (shows 400 PPM threshold)
â€¢ Past alert still shows: 500 PPM threshold (what was active then)
